Plan Mode
Qué es

Claude presenta un plan de acción completo antes de ejecutar cualquier cambio. Puedes leerlo, corregirlo y aprobarlo antes de que toque un solo archivo.

Ideal para tareas grandes, cambios en múltiples archivos u operaciones difíciles de revertir.

Cómo activarlo
Shift + Tab Alterna el plan mode on/off desde la terminal
También puedes pedir a Claude: "antes de hacer nada, muéstrame el plan"
Flujo recomendado
01  Activa plan mode  ──  Shift+Tab
02  Escribe tu petición en lenguaje natural
03  Claude muestra el plan — revísalo
04  Aprueba, corrige o cancela antes de ejecutar
05  Desactiva plan mode para ajustes menores
Atajos de teclado
Escape Cancela la generación en curso o vuelve al prompt anterior
Ctrl + C Interrumpe inmediatamente lo que Claude está ejecutando
↑ / ↓ Navega el historial de prompts de la sesión actual
Ctrl + R Búsqueda inversa en el historial — escribe para filtrar prompts anteriores
Tab Autocompletado de comandos y rutas de archivo
Shift + Tab Alterna el plan mode on/off
Ctrl + L Limpia la pantalla de la terminal (equivalente a /clear)
Ctrl + D Cierra la sesión de Claude Code y sale del CLI
Modo multi-línea
\ + Enter Continúa el prompt en la línea siguiente — para instrucciones largas
Ctrl + J Alternativa para insertar salto de línea en el prompt sin enviarlo
Extensión VS Code
Instalación

Abre VS Code → Extensions (Ctrl+Shift+X) → busca "Claude Code" → Install.

O desde la terminal:

code --install-extension anthropic.claude-code
Qué añade la extensión
Estado en tiempo real — ves qué está haciendo Claude Code sin salir del editor
Diff visual — revisa cada cambio en archivos con el visor de diferencias de VS Code
Historial de sesión — navega los cambios generados en la sesión actual
Terminal integrada — lanza Claude Code desde VS Code con un panel dedicado
La terminal sigue siendo el centro de trabajo. La extensión complementa la experiencia visual.

Cómo funcionan las Skills
Qué es una skill

Un archivo Markdown en .claude/skills/ que se convierte en un comando personalizado. Cuando escribes /nombre, Claude ejecuta exactamente lo que está en ese archivo.

El contenido es lenguaje natural — ya sabes cómo escribirlo. Es una especificación para un comando.

Skills de proyecto
.claude/skills/nombre.md
Solo funcionan dentro del proyecto. Si subes el repo, cualquier colaborador las hereda automáticamente.
Skills globales
~/.claude/skills/nombre.md
Disponibles en cualquier proyecto que abras. Son tu caja de herramientas personal de Claude Code.
Skills del proyecto — AI Spec Builder
/revisar Lee el CLAUDE.md y la spec, revisa los archivos modificados y lista inconsistencias entre código y especificación
/commit Revisa los cambios staged y genera un commit con formato Conventional Commits automáticamente
/explicar Resume en lenguaje no técnico qué hizo Claude en la última sesión y lo relaciona con la spec
/spec-check Analiza las 6 secciones de la spec buscando contradicciones e inconsistencias antes de construir
Skills globales del curso
/nuevo-proyecto Inicializa cualquier proyecto con la estructura spec-first: carpetas, CLAUDE.md y borrador de las 6 secciones
/daily Lee el git log y el CLAUDE.md, genera un resumen del día: qué se completó, qué sigue y próximo paso
Workflow completo con Skills
Antes de empezar
  /spec-check  →  valida la spec
Mientras construyes
  /revisar  →  compara con la spec
  /explicar  →  entiende lo generado
Al terminar la sesión
  /commit  →  commit automático
  /daily  →  resumen del día
Proyecto nuevo
  /nuevo-proyecto  →  spec-first listo
Crea tus propias skills
1. Identifica una instrucción que repites en cada sesión
2. Crea el archivo en .claude/skills/nombre.md
3. Escribe la instrucción en lenguaje natural — como le hablarías a Claude
4. Usa /nombre desde la terminal
Recuerda

Una skill es una especificación para un comando. Ya sabes escribir especificaciones — eso es exactamente lo que aprendiste en el Módulo 1.
