## Ejercicio 1 — Comparte y recoge feedback
**Dificultad:** Básico
**Tiempo estimado:** 15 minutos

Tu AI Spec Builder está en internet. Ahora comprueba que funciona para alguien que no seas tú.

**Instrucciones:**
1. Copia la URL de tu AI Spec Builder desplegado en Vercel
2. Envíala a 2-3 personas (amigos, colegas, familiares — no importa si son técnicos o no)
3. Pídeles que:
   - Escriban una idea de producto (la que sea)
   - Generen la especificación
   - Te digan: ¿Entendieron qué hacía la herramienta? ¿El resultado les fue útil? ¿Algo les confundió?
4. Documenta el feedback:

| Persona | Idea que probó | ¿Entendió la herramienta? | Feedback sobre el resultado | ¿Algo confuso? |

| | | | | |
| | | | | |

**Reflexión:** El feedback real de usuarios reales es irreemplazable. ¿Qué descubriste que no habías pensado?

## Ejercicio 2 — Aplica el sistema a tu propia idea
**Dificultad:** Intermedio
**Tiempo estimado:** 45-60 minutos

Este es el ejercicio más importante del curso. Usa el sistema spec-first completo para arrancar tu propio proyecto.

**Instrucciones:**
1. **Elige tu idea** — Un producto, herramienta o app que quieras construir
2. **Escribe la spec** — Usa la Plantilla Spec-First con las 6 secciones (ya la practicaste en el Módulo 1)
3. **Crea el CLAUDE.md** — Resume tu spec en un CLAUDE.md
4. **Inicializa el proyecto** — Usa Claude Code para crear la estructura del proyecto
5. **Construye el primer componente** — Pide a Claude Code que cree la primera pieza funcional

**No necesitas terminarlo hoy.** El objetivo es arrancar con el sistema y verificar que funciona.

**Entregable:** Un proyecto inicializado con CLAUDE.md, spec completa, y al menos un componente funcionando.

**Reflexión:** ¿El proceso se sintió más ordenado que proyectos anteriores? ¿En qué momento la spec te ahorró una decisión que habrías tomado al azar?
